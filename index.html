How It Works

This application provides military-grade encryption entirely in your browser. Your files never leave your device during the encryption/decryption process, ensuring maximum privacy and security.

üîë Encryption Process

  1. Key Derivation: Your password is converted into a 256-bit encryption key using PBKDF2 with 100,000 iterations and a random salt. This makes brute-force attacks computationally infeasible.
  2. Compression (Optional): Files are compressed using gzip via the Web Compression Streams API, reducing file size significantly for text-based content.
  3. AES-256-GCM Encryption: Your data is encrypted using AES-256 in GCM mode, which provides both confidentiality and authentication. A random 12-byte IV (Initialization Vector) ensures each encryption is unique.
  4. Signature (Optional): If enabled, your name/identifier is separately encrypted with its own IV using the same key, allowing recipients to verify the sender.
  5. .crypt File Format: All components (salt, IV, encrypted data, signature) are packaged into a JSON file with Base64 encoding for easy transmission.

üîì Decryption Process

  1. File Parsing: The .crypt file is read and parsed to extract salt, IV, encrypted data, and optional signature.
  2. Key Derivation: The same PBKDF2 process recreates the encryption key from your password and the stored salt.
  3. Signature Verification: If present, the signature is decrypted first to show you who sent the file.
  4. Decryption: AES-GCM decrypts the data. If the password is wrong or data is tampered with, decryption fails immediately.
  5. Decompression: If the file was compressed, it's decompressed back to its original form.
  6. Download: Your original file is reconstructed and downloaded with its original filename.

  • Client-Side Only: All operations happen in your browser. No data is uploaded to any server.
  • AES-256-GCM: Industry-standard encryption used by governments and militaries worldwide.
  • Authenticated Encryption: GCM mode prevents tampering by detecting any modifications to encrypted data.
  • Random Salt & IV: Each encryption uses unique random values, preventing pattern analysis.
  • Strong Key Derivation: 100,000 PBKDF2 iterations make password cracking extremely difficult.
  • Verification Signatures: Optional sender identification helps prevent social engineering attacks.

üí° Best Practices

  • Use strong, unique passwords (minimum 12 characters with mixed case, numbers, symbols)
  • Share passwords through a different channel than the encrypted file

  • Use verification signatures when sending files to verify authenticity
